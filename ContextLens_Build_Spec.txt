# ContextLens: Build Specification & Architecture

ContextLens is an autonomous coding observability platform that tracks development "episodes," logs AI interactions, and provides Gemini-powered insights into code evolution.

## 1. System Architecture

The system consists of three primary components:
- **VS Code Extension**: The primary data collector and user interface for developers.
- **Backend (API)**: An Express-based service managing authentication, data persistence, and Gemini integrations.
- **Dashboard (Web)**: A React-based visualization interface for reviewing project history and branch summaries.

```mermaid

    VSCode[VS Code Extension] -->|Auth/Sync| Backend[Express API]
    VSCode -->|Autonomous Watchers| GitHead[.git/HEAD]

```

## 2. Component Details

### A. VS Code Extension (`/vscode-extension`)
- **`AuthManager`**: Manages OIDC flow with Google. 
    - Uses a custom protocol `vscode://` to capture tokens from the web-based login.
    - Implements custom token exchange to stay within the Firebase Auth sandbox while maintaining extension-level security.
- **`EpisodeStore`**: Maintains workspace-level state for the active task ("Episode"). 
    - Maps git remotes to Firestore `projects`.
    - Detects active branches and manages the `activeEpisodeId` in local persistent storage.
- **`SyncEngine`**: A robust background worker with an offline-first queue. 
    - Prevents data loss by persisting the queue to `mementos` if the API is unreachable.
    - Uses exponential backoff for retries.
- **`Watchers`**:
    - `watchBranch`: Listens to `.git/HEAD` changes. Closes current episode and starts a new one for the target branch.
    - `watchCommits`: Listens to `.git/COMMIT_EDITMSG`. Automatically logs the commit as a "Milestone" within the current episode.
    - `watchFileSaves`: Tracks changed files for diff context.
- **`GitContext`**: Utility to extract branch, diffs, and "TODO" markers. 
    - Sanitizes diffs to remove sensitive information before sending to the AI.

### B. Backend API (`/src`)
- **`firebase.js`**: Centralized Firebase Admin SDK initialization. Manages Firestore and Auth instances.
- **`routes/api.js`**: Core endpoints:
    - `POST /projects/create`: Uses git remote URL as a unique key to resolve or create project records.
    - `POST /episodes/create`: Atomic creation of task sessions.
    - `POST /calls/log`: Ingests AI interactions, mapping them to the current episode, branch, and file context.
    - `POST /episodes/explain`: Orchestrates Gemini to analyze diffs.
- **`services/ai.js`**: Vertex AI / Gemini 1.5 Pro wrapper.
    - Implements prompt templates for code risk analysis and architectural summary.
- **`middleware/auth.js`**: Verifies Firebase ID tokens on every incoming request from the extension or dashboard.

### C. Web Dashboard (`/contextlens-dashboard`)
- **React + Vite + Tailwind**: High-performance, premium UI.
- **`pages/Dashboard`**: Visualizes the project hierarchy.
- **`pages/EpisodeDetail`**: A chronological timeline of developer activity, including commits, file saves, and AI interactions.

## 3. Core Workflows

### 1. Authentication & Token Exchange
1. Developer clicks "Login" in VS Code.
2. Extension opens `https://contextlens-backend-001.web.app/api/auth/login?callback=vscode://89Aman.contextlens/auth`.
3. User signs in with Google on the web page.
4. Backend verifies the Google ID token and issues a **Custom Firebase Token**.
5. The web page redirects back to VS Code via the custom URI protocol.
6. Extension exchanges the Custom Token for a long-lived **ID Token** via the REST API.

### 2. Autonomous Episode Lifecycle
1. **Detection**: `Watcher` notices a branch change.
2. **Resolution**: `EpisodeStore` looks up the project via git remote.
3. **Creation**: If the branch has no active episode, a new one is created via `POST /episodes/create`.
4. **Auto-Closure**: When the branch is merged or abandoned (detected via git state), the episode is marked as `closed`.

### 3. Context-Aware AI Logging
1. Developer prompts an AI (e.g., Gemini Chat).
2. The extension captures the prompt.
3. **Enrichment**: The extension attaches:
    - The full git diff since the last milestone.
    - The content of the active file.
    - Any recent "TODO" comments found in the codebase.
4. **Ingestion**: The data is batched and sent via `SyncEngine` to `POST /calls/log`.

## 4. Advanced Technical Deep-Dive

### Data Normalization
The system treats the **Git Remote** as the source of truth for Project Identity. This ensures that multiple developers working on the same repo are mapped to the same `ContextLens` project, even if their local folder names differ.

### Resilience Patterns
- **Sync Engine Persistence**: If VS Code is closed while logs are pending, they are saved to global storage and resumed on the next launch.
- **PII Redaction**: All logs pass through a regex-based redaction layer to scrub potential secrets (API keys, passwords) before being stored or sent to Gemini.

### AI Reasoning Layer
ContextLens uses Gemini 1.5 Pro with a temperature of `0.2` for deterministic summaries. It specifically looks for "Architectural Drift" — where code changes deviate from the established patterns in the project history.

## 5. Development Guidelines
- **Offline First**: All telemetry from the extension must pass through the `SyncEngine`.
- **Zero Friction**: Prefer automatic event detection (watchers) over manual logging where possible.
- **Context Rich**: Always include git diffs and file paths in AI logs to improve downstream analysis.

## 6. Utilities & Tooling

### A. ContextLens CLI (`/contextlens-cli.js`)
The project includes a standalone Node.js CLI tool for manual operations and project maintenance.
- **`init`**: Bootstraps the ContextLens directory structure in a new workspace.
- **`sync`**: Manually triggers the `SyncEngine` to push local memento data to Firestore.
- **`config`**: Manually set `firebaseProject` or `syncInterval` tokens.
- **`status`**: Validates the project setup and reports version info.

### B. Migration Scripts (`/migrate-demo-to-real.js`)
A utility script used during the transition from the "Demo" anonymous environment to the "Real" authenticated environment.
- Performs recursive copying of `projects`, `episodes`, and `calls` sub-collections.
- Maps `DEMO_UID` documents to `REAL_UID` destinations while preserving document IDs for link integrity.

## 7. Development Guidelines
- **Offline First**: All telemetry from the extension must pass through the `SyncEngine`.
- **Zero Friction**: Prefer automatic event detection (watchers) over manual logging where possible.
- **Context Rich**: Always include git diffs and file paths in AI logs to improve downstream analysis.
- **Atomic Commits**: Follow conventional commits (feat, fix, docs, refactor) to maintain a clean history for Gemini analysis.
