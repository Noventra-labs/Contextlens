ContextLens - Builder Spec 3 - Web Dashboard and Demo Polish
Version: 3.0 | Stack: Vite + React + TypeScript + Tailwind CSS + Firebase Hosting
Replaces all previous Next.js dashboard specs.

=============================================================
ROLE
=============================================================

You are responsible for building the full web dashboard and all
demo-facing polish for ContextLens.

Product Context:
ContextLens is an AI workflow memory layer for coding. It captures
AI-assisted coding sessions as episodes and visualizes them as a
searchable timeline with diffs, prompts, responses, and summaries.

Your Ownership:
- Vite + React dashboard (replaces Next.js — too heavy for this scope)
- Project overview UI
- Branch navigation UI
- Episode timeline UI
- Episode detail pages
- Branch summary pages
- Search and filter UX
- Copy/export UX
- Visual polish for judges/demo

Do Not Build:
- VS Code extension commands
- Git capture logic
- Backend business logic
- Firebase Functions or Gemini prompts
- Firestore security rules (already deployed)
- Auth token handling inside VS Code

Core Objectives:
- Make the product legible and impressive in under 2 minutes.
- Let users inspect project history through episodes and calls.
- Present Explain Diff and Branch Summary results beautifully.
- Make the dashboard feel premium and trustworthy.

SECTION 1: TECH STACK
=============================================================

Framework:       Vite + React 18 + TypeScript
Styling:         Tailwind CSS v3
Routing:         react-router-dom v6
Auth:            Firebase Auth — Google Sign-In (firebase/auth)
Database reads:  Firestore client SDK (firebase/firestore)
API calls:       fetch() with Firebase ID token in Authorization header
Diff viewer:     react-diff-viewer-continued
Hosting:         Firebase Hosting
State:           React Context + useState/useEffect
Icons:           lucide-react
Fonts:           Geist Sans (body) + Geist Mono (code/labels)

Why not Next.js:
- This is a pure client SPA reading from Firestore + calling HTTP endpoints.
- No SSR, no server components, no getServerSideProps needed.
- Vite is faster to develop with, lighter to build, and deploys identically
  to Firebase Hosting with a single: npm run build && firebase deploy

SECTION 7: FIRESTORE HOOKS (src/lib/firestoreHooks.ts)
=============================================================

Build these custom hooks. Each returns { data, loading, error }.

--- useProjects(uid: string) ---
  collection: users/{uid}/projects
  query: orderBy("updatedAt", "desc")
  mode: onSnapshot (real-time)
  returns: Project[]

--- useEpisodes(uid: string, projectId: string) ---
  collection: users/{uid}/projects/{projectId}/episodes
  query: orderBy("startedAt", "desc")
  mode: onSnapshot (real-time)
  returns: Episode[]
  Note: use this on ProjectPage so new episodes from VS Code appear instantly

--- useEpisodesByBranch(uid, projectId, branchName) ---
  collection: users/{uid}/projects/{projectId}/episodes
  query: where("branchName", "==", branchName), orderBy("startedAt", "asc")
  mode: onSnapshot
  returns: Episode[]

--- useEpisode(uid, projectId, episodeId) ---
  document: users/{uid}/projects/{projectId}/episodes/{episodeId}
  mode: onSnapshot (live — so callCount updates live when extension logs)
  returns: Episode | null

--- useCalls(uid, projectId, episodeId, enabled: boolean) ---
  collection: users/{uid}/projects/{projectId}/episodes/{episodeId}/calls
  query: orderBy("createdAt", "asc")
  mode: getDocs (one-time fetch, not live)
  enabled: only fetch when episode card is expanded (lazy load)
  returns: Call[]

Helper to convert Firestore Timestamp to Date:
  const toDate = (ts: any): Date => ts?.toDate?.() ?? new Date(ts)

SECTION 8: BACKEND API CONTRACTS (src/lib/api.ts)
=============================================================

Base URL: import.meta.env.VITE_API_BASE_URL
All requests need: Authorization: Bearer {idToken}

Helper — get token:
  async function getToken(): Promise<string> {
    const user = auth.currentUser
    if (!user) throw new Error("Not authenticated")
    return user.getIdToken()
  }

Helper — authenticated POST:
  async function post<T>(path: string, body: object): Promise<T> {
    const token = await getToken()
    const res = await fetch(`${import.meta.env.VITE_API_BASE_URL}${path}`, {
      method: "POST",
      headers: {
        Authorization: `Bearer ${token}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify(body),
    })
    if (res.status === 401) throw new Error("UNAUTHORIZED")
    if (!res.ok) throw new Error(`API_ERROR_${res.status}`)
    return res.json()
  }

--- explainDiff(projectId, episodeId) ---
  POST /episodes/explain
  Body: { projectId, episodeId }
  Returns: ExplainDiffResult { summary, risks[], checks[] }

  Smart cache check BEFORE calling:
    If episode.explainDiffSummary is already set in Firestore,
    display it directly — do not call the API again.
    The backend writes the result back to Firestore after first call,
    so Firestore is the cache layer.

--- branchSummary(projectId, branch) ---
  POST /branches/summarize
  Body: { projectId, branch }
  Returns: BranchSummaryResult { pr_summary, key_changes[], review_risks[] }

--- search(query, filters) ---
  POST /search
  Body: {
    query: string,
    filters: {
      branchName?: string,
      filePath?: string,
      dateFrom?: string,   // ISO date string
      dateTo?: string      // ISO date string
    }
  }
  Returns: {
    episodes: Array<{
      episodeId, projectId, label, branchName, callCount, startedAt
    }>,
    calls: Array<{
      callId, episodeId, intentTag, activeFilePath, createdAt
    }>
  }

Error handling rules:
  - UNAUTHORIZED: sign user out → navigate("/login")
  - API_ERROR_500: show inline error with retry button
  - Network failure: "Connection error. Check your network."
  - Never crash the UI on an API failure — show error state in component

--- DiffViewer ---
Props: diff: string
Library: react-diff-viewer-continued

Implementation:
  - Parse unified diff string to split into oldValue / newValue
    (split on @@ hunk headers, reconstruct old and new sides)
  - Render with:
    <ReactDiffViewer
      oldValue={oldValue}
      newValue={newValue}
      splitView={false}
      useDarkTheme={true}
      hideLineNumbers={false}
    />
  - Wrap in a bordered card with "Diff" header label
  - Collapsed by default if diff > 30 lines — show "Show diff ({n} lines)" toggle
  - If diffSnapshot is empty string or null: render nothing (skip entirely)

SECTION 14: LOADING AND ERROR STATE RULES
=============================================================

Every data-fetching component MUST handle all three states:

Pattern:
  if (loading) return <SkeletonCard />
  if (error) return <ErrorMessage message={error} onRetry={refetch} />
  if (data.length === 0) return <EmptyState ... />
  return <DataComponent data={data} />

Never:
  - Show an empty list while loading
  - Crash or show blank screen on API errors
  - Show raw error objects to the user
