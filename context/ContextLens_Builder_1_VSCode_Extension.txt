ContextLens - Builder Spec 1 - VS Code Extension

Role
You are responsible for building the full VS Code extension experience for ContextLens.

Product Context
ContextLens is an AI workflow memory layer for coding. It captures AI-assisted coding sessions as structured episodes linked to git context, diffs, prompts, responses, and summaries.

Your Ownership
- VS Code extension in TypeScript
- Activity bar icon
- Sidebar panel 1: State Header
- Sidebar panel 2: Gemini Chat webview
- Status bar item
- Command palette actions
- Local extension state
- Git context capture
- Calling backend APIs
- Retry queue for failed requests
- SecretStorage usage for auth/session state

Do Not Build
- Web dashboard UI
- Firestore schema design decisions beyond consuming shared contracts
- Backend business logic
- Gemini summarization service internals
- JetBrains plugin
- GitHub PR integration
- Team collaboration

Primary User Outcomes
- User can create, switch, and close episodes inside VS Code.
- User can send an AI request with optional intent tag.
- Extension automatically captures branch, active file, diff snapshot, and TODO/FIXME markers.
- User can trigger Explain Diff and Branch Summary actions.
- User can open dashboard from the extension.

Required Surfaces
1. Activity Bar Entry
- Add ContextLens icon to VS Code activity bar.
- Clicking opens ContextLens sidebar.

2. Sidebar Panel 1: State Header
Must show:
- Active episode name
- Current branch
- AI call count
- AI-owned files
- You-owned files
- Manual note field or note action
Must support actions:
- New Episode
- Close Episode
- Open Dashboard
- Explain Current Diff

3. Sidebar Panel 2: Gemini Chat Webview
Must include:
- Prompt input
- Optional intent tag field
- Chat history for current episode
- Send button
- Retry button
Behavior:
- Every request captures git context before sending.
- Every successful request increments visible call count.
- Chat history should belong to the current episode.

4. Status Bar
Format:
- ContextLens: {episode} · {n} calls
Behavior:
- Clicking focuses ContextLens panel.

5. Command Palette Commands
Implement:
- ContextLens: New Episode
- ContextLens: Close Episode
- ContextLens: Explain Diff
- ContextLens: Summarize Branch
- ContextLens: Open Dashboard

Required Modules
- extension.ts
- auth.ts
- episodeStore.ts
- gitContext.ts
- chatViewProvider.ts
- stateTreeProvider.ts
- apiClient.ts
- telemetry.ts
- redaction.ts

Git Capture Requirements
- Detect current branch name.
- Capture diff snapshot at request time.
- Capture active file path.
- Parse TODO/FIXME/HACK markers from active file.
- If workspace is not a git repo:
  - disable diff capture
  - allow manual logging
  - show warning state

Local State Requirements
- activeProjectId
- activeEpisodeId
- authSession
- draftIntent
- lastDiffHash
- queuedRequests

Auth / Session Requirements
- Use secure session handling.
- Persist auth/session state across extension reloads if possible.
- Use VS Code SecretStorage for tokens/session material.

Expected Behaviors
- Creating an episode stores active episode locally.
- Closing an episode clears active episode and updates UI.
- Sending a prompt packages prompt + git context + episode info and calls backend.
- Failed calls can be retried.
- Duplicate diff spam should be reduced using lastDiffHash if useful.

Acceptance Criteria
- User can create an episode in VS Code.
- User can send a prompt from the extension.
- Extension captures branch, file, diff, and intent tag.
- Call count updates after successful response.
- User can trigger Explain Diff from the extension.
- Extension remains usable even if backend is temporarily unavailable.

Implementation Notes
- Build only for VS Code MVP.
- Keep UI simple but polished.
- Prefer reliable flows over fancy abstraction.
- Use the shared backend contract from the backend builder.
- Keep code modular and hackathon-friendly.

Deliverables
- Working VS Code extension
- Sidebar tree view and chat webview
- Command palette integration
- Status bar integration
- API integration hooks using shared contracts
- Minimal setup/run instructions
