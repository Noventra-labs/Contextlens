# ContextLens MCP Troubleshooting

This guide helps resolve common issues encountered while setting up or using the ContextLens Model Context Protocol.

---

## 🔍 Connection Issues

### "ContextLens VS Code Extension is not running"
The bridge was able to launch, but it cannot connect to the local extension server.
* **Solution**: Ensure VS Code is open and that the ContextLens extension is active. The extension starts the helper HTTP server on port `3012`.

### "Unauthorized — invalid or missing MCP secret"
The client token sent by the bridge does not match the token generated by the VS Code extension.
* **Why**: Tokens expire and rotate every 30 minutes, or a new token was generated when VS Code restarted.
* **Solution**: The bridge auto-refreshes by re-reading `.mcp-secret.json` on a 401 error. If this fails, restart your AI client (Claude Desktop / Cursor) to force it to re-run the bridge initialization.

### "Address already in use (EADDRINUSE)"
The VS Code extension helper server cannot bind to port `3012`.
* **Solution**: Another process is using port `3012`. You can find and terminate it:
  * **Windows**: `netstat -ano | findstr :3012`
  * **macOS/Linux**: `lsof -i :3012`

## 🛠️ Configuration Issues

### "contextlens command not found" (Claude / Cursor)
The AI client tried to launch `contextlens mcp run` but the executable was not found in the environment PATH.
* **Solution 1**: Ensure you installed the CLI globally: `npm install -g @contextlens/cli`.
* **Solution 2**: Configure the absolute path directly in your config:
  ```json
  "args": ["/absolute/path/to/global/node_modules/@contextlens/cli/dist/index.js"]
  ```

### Configuration paths on Windows
Paths in JSON strings must escape backslashes:
* **Incorrect**: `C:\Users\name\mcp-bridge.js`
* **Correct**: `C:\\Users\\name\\mcp-bridge.js` or use forward slashes `C:/Users/name/mcp-bridge.js`.

## 📊 Rate Limiting (HTTP 429)

### "Rate limit exceeded" or "Rate limit exceeded for tool..."
You have made too many requests in a short window.
* **Standard limits**: 120 requests/minute.
* **Expensive limits**: 10 requests/minute (for `explain_diff`, `explain_past_changes`, `search_context`).
* **Solution**: Wait for the rate-limiting window to reset (typically 60 seconds). You can check remaining requests in the response headers:
  * `X-RateLimit-Remaining`
  * `X-RateLimit-Limit`
  * `Retry-After`

## 🐞 Diagnostics Tool

You can check the health of all components at any time by running:
```bash
contextlens mcp doctor
```

This runs diagnostics on:
1. Node.js version.
2. Extension path existence.
3. Extension server port `3012` connectivity.
4. Active token file.
